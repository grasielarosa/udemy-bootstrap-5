Sistema Grid

O bootstrap trabalha com um sistema de grids que vai de zero até doze colunas e nós controlamos a disposição dos elementos através delas.

Por exemplo, um menu costuma ocupar toda a lateralidade do site, então um nenu usaria as doze colunas do grid.

Em um site de notícias, usariamos metade dessas colunas para um título e a outra metade para uma imagem


Responsividade - Breakpoints no Bootstrap

X-Small; sem infixo; resolução <576px.

Small; sm; resolução >=576px.

Medium; md; resolução >=768px.

Large; md; resolução >=992px.

Extra Large; lg; resolução >=1200px.

Extra extra Large; xxl; resolução >=1400px.

Ps.: essas resoluções são na largura.

Container

Tudo que está na página deve estar dentro de containers, que servem como uma estrutura para definir o tamanho ou o espaço máximo que vamos ocupar na página.

Conatiner Padrão - default

Quando não há sufixo ou infixo (sm, md, etc), se considera que ele será aplicado desde o mais pequeno (X-small). É a classe container aplicada sem nada e que geralmente é usada nas páginas.

O próprio container já tem vários comportamentos predefinidos para MediaQueries

Container com Breakpoints

São os containers com os infixos de breakpoints do módulo anterior. Se escrevem com a palavra container, hífen e logo o infixo escolhido.

Ao adotar os breakpoints. Alguns elementos ficam com resolução de 100% (bo breakpoint para baixo) e outros para ficarem com a resolução fixa (do breakpoint para cima).

Container fluid

Adota sempre 100% da largura da tela, não importa a resolução.
